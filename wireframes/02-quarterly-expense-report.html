🎨 Design Notes & Decisions

✨ NEW: Multiple Input Methods

Why: Users with many expenditures (10+) find manual entry tedious and error-prone

Options:

  • Manual Entry: Good for 1-5 items, direct inline editing
  • CSV Upload: Best for 10+ items, prepare offline in Excel/Sheets
  • Bulk Paste: Middle ground - copy from existing spreadsheet, paste directly

User Flow: Choose method → Input data → System validates → User reviews → Confirm import

Time Savings: CSV/Paste methods reduce 20-minute manual entry to ~2 minutes

CSV Template & Validation

Template Download: Pre-formatted CSV with example data and correct headers

Validation Rules:

  • Date format: YYYY-MM-DD, within quarter range
  • Amount: Valid number, >$0
  • Required fields: Date, Official, Payee, Purpose, Amount
  • Is Estimate: "Yes"/"No" (case insensitive)

Error Handling: Show row numbers with errors, allow inline fix or re-upload

Preview Before Import

Why: User confidence - see exactly what will be imported before confirming

Features: Color-coded validation (green=valid, orange=warning, red=error), edit before confirm, warnings don't block import

Hybrid Workflow Support

Scenario: User uploads CSV with 15 items, realizes they forgot 1 item

Solution: After CSV import, switch to Manual mode and add additional rows

Implementation: All methods populate same underlying data structure, seamlessly combine

Summary + Itemized Approach

Why: Matches ordinance structure (total + itemization for >$50), clear two-part flow

Calculation: Grand total auto-calculates from summary fields

Mobile Considerations

  • Manual table difficult on mobile - switch to card layout
  • CSV upload works great on mobile (prepare on desktop, upload from mobile)
  • Bulk paste less ideal for mobile (typing in textarea)

Future Enhancements (Phase 2)

  • Excel file upload (.xlsx) - same validation as CSV
  • Save user's template with their formatting preferences
  • Auto-complete public official names from database
  • Duplicate detection (same date + official + amount)
  • Import from previous quarter (copy last quarter's expenses)

API Integration (Phase 3)

For large employers/firms: Direct API submission of expense data

Format: JSON POST to /api/expense-reports/{quarter}

Benefit: Integrate with their existing accounting systems
