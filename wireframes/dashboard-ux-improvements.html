üé® Design Notes & Rationale

1. Removed "Exemption Checker" from Dashboard

Why: Once logged in, the user has already determined they need to register. This tool is for pre-registration discovery only.

Impact: Reduces cognitive load, removes redundant decision-making

2. Moved "Register" to User Menu

Why: Registration is a ONE-TIME or very rare action (update info annually). It shouldn't have equal visual weight to weekly/quarterly tasks.

New Location: User dropdown ‚Üí "Update Registration" (settings-like action)

3. Visual Hierarchy by Frequency

Tier 1 (Large cards): Hour Tracking (weekly) & Quarterly Reports (every 3 months)

Tier 2 (Small cards): Violations (as-needed), Public search (occasional)

Tier 3 (Menu items): Update Registration (rare), Settings

Principle: Screen real estate proportional to task frequency

4. Contextual Alerts for Critical Actions

Not Registered: Full-width urgent alert with deadline countdown

Near Threshold: Warning when approaching 10-hour limit (proactive UX)

Report Due Soon: Inline deadline reminders with urgency badges

Principle: Show what matters NOW, hide what doesn't

5. "My Work" Menu Reordered

Before: Register, My Reports, My Violations (alphabetical?)

After: Hour Tracking, Quarterly Reports, My Violations (by frequency)

Bonus: Added status badges (8.5/10, Due Oct 15) for at-a-glance info

6. Progressive Disclosure

Not Registered: Disable/gray out actions that require registration first

Registered: Show full dashboard with all tools available

Principle: Don't show options users can't use yet
