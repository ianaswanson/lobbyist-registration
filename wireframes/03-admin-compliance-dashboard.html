üé® Design Notes & Decisions

Dashboard Layout: 2/3 + 1/3 Column

Why: Main content (deadlines, registrations, reports) on left, contextual actions/info on right

Benefit: Scan left for work items, right for quick actions and monitoring

Alert-Driven Interface

Why: Admin needs to prioritize urgent items (overdue reports, pending registrations)

Color Coding: Red = urgent/overdue, Orange = needs attention, Blue = informational, Green = positive action

In-Line Actions vs. Drill-Down

Chosen: Quick actions (Approve/Reject/Review) directly in tables for high-frequency tasks

Alternative: Click to detail page for every action (slower workflow)

Stats Cards at Top

Why: At-a-glance system health, trend indicators (up/down arrows)

Future: Click stat card to filter view (e.g., click "Active Lobbyists" to see list)

Days Pending/Late Badges

Why: Visual urgency indicator, helps prioritize review queue

Thresholds: 0-3 days (yellow), 4-7 days (orange), 8+ days (red)

Bulk Actions

Implementation: "Send Bulk Reminders" button ‚Üí modal with filters (overdue only, specific quarter, etc.)

Why: Admin efficiency for common tasks (reminder emails before deadline)

Recent Activity Feed

Why: Awareness of system activity, especially useful for multi-admin teams

Links to: Full audit log with filtering/search

Accessibility Considerations

  • Color not sole indicator (also text/icons)
  • Keyboard navigation for table actions
  • Screen reader labels for icon buttons
  • Focus management for modal workflows

Missing Features (Future)

  • Search/filter functionality on tables
  • Export to CSV for reporting
  • Batch approval workflow
  • Dashboard customization (admin preferences)
  • Real-time notifications (WebSocket)
