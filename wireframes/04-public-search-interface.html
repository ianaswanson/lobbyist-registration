üé® Design Notes & Decisions

Search-First Interface

Why: Primary user goal is finding specific lobbyists/employers/subjects

Pattern: Large search bar + collapsible advanced filters

List View vs. Card View

Chosen: List view (more information density, easier scanning)

Alternative: Card view (better for mobile, more visual)

Solution: Provide toggle button (currently showing list, can switch to cards)

Result Item Information Hierarchy

Priority: Name (largest) ‚Üí Employer ‚Üí Subjects ‚Üí Spending ‚Üí Actions

Right-aligned spending: Draws eye, enables quick spending comparison

Advanced Filters

Why collapsible: Keeps interface clean, power users can expand

Filters chosen: Based on common search patterns (employer type, subject, year, spending, quarter)

Future: Save filter presets, filter by Board member (who received expenditures)

Sort Options

Default: Alphabetical by name

Other options: Registration date, spending amount (enables "top spenders" view)

Export Functionality

Why important: Media/researchers want to analyze data in spreadsheets

Implementation: CSV export of current search results with applied filters

Mobile Responsiveness

  • Search bar full-width on mobile
  • Advanced filters stack vertically
  • Results switch to card layout on mobile
  • Spending moved below name on mobile
  • Pagination simplified (prev/next only)

Accessibility Considerations

  • Search landmark role
  • Form labels properly associated
  • Keyboard navigation through results
  • Screen reader announces result count
  • Focus management for filter toggle

Performance Considerations

Search: Debounce quick search (300ms), instant advanced filters

Pagination: 25 results default (balance between scrolling and page loads)

Future: Infinite scroll option for power users

Missing Features (Future Phase)

  • Save searches (user accounts)
  • Email alerts for new registrations matching criteria
  • Spending trend charts on search results
  • Compare multiple lobbyists side-by-side
  • Map view (geocode addresses, show on map)
  • API access for developers
