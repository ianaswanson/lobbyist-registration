About This Demo

This walkthrough demonstrates a lobbyist registration system built to meet Multnomah County's Government Accountability Ordinance requirements (effective July 1, 2026). The system supports five distinct user workflows: public transparency, lobbyist registration and reporting, employer expense reporting, board member calendar posting, and administrative compliance monitoring.

Demo Time: 15-20 minutes for full walkthrough, or 8-10 minutes for key highlights only.

Test Data: All accounts have pre-loaded data based on realistic Portland lobbying scenarios. The "Rule of 3" pattern means 3 entities of each type, with 3 reports each, and 3 line items per report.

Demo Flow

Five user workflows demonstrate how different stakeholders interact with the system. The demo follows the data lifecycle: from filing to review to public transparency.

Step 2 Key Feature: Calendar File Import

Start a new Q2 2025 submission. Show the calendar import options:

  • Manual Entry: Add individual calendar entries via form fields.
  • CSV Upload: Upload a spreadsheet with meeting dates, titles, and attendees.

Technical Implementation

Technology Stack

  • Framework: Next.js 15 (App Router) + React 19
  • Language: TypeScript (strict mode)
  • Database: PostgreSQL (Cloud SQL) + Prisma ORM
  • Authentication: NextAuth.js with session management
  • UI: Tailwind CSS + shadcn/ui components
  • Deployment: Google Cloud Run (containerized)
  • CI/CD: GitHub Actions + Cloud Build Triggers
  • Monitoring: Sentry error tracking, Cloud Run metrics

Infrastructure

  • Dev Environment: Auto-deploy from 'develop' branch
  • Prod Environment: Manual approval from 'main' branch
  • Database: Cloud SQL PostgreSQL (shared across instances)
  • Secrets: Google Cloud Secret Manager
  • Security: Dependabot for dependency updates
  • Accessibility: WCAG 2.1 AA compliant
  • Cost: ~$84/month infrastructure + database
